## 1단계: 재료 등급 · 드롭 · 경험치 서

### 1-A. State 구조 변경 (`js/state.js`)

**변경 전:**
```js
materials: 5,
```

**변경 후:**
```js
materials: { common: 5, advanced: 0, rare: 0, legendary: 0 },
```

**마이그레이션 (loadState 직후):**
```js
if (typeof State.materials === 'number') {
  State.materials = { common: State.materials, advanced: 0, rare: 0, legendary: 0 };
}
```

**재료 함수 교체:**
```js
// 추가
function addMaterial(grade, amount) { State.materials[grade] += amount; }

// 지출: 재료 오브젝트 { common, advanced, rare, legendary } 전달
function spendMaterials(cost) {
  for (const [grade, amt] of Object.entries(cost)) {
    if ((State.materials[grade] || 0) < amt) return false;
  }
  for (const [grade, amt] of Object.entries(cost)) {
    State.materials[grade] -= amt;
  }
  return true;
}
```

### 1-B. 드롭 시스템 변경 (`js/combat.js`)

**파견 dispatch 초기화:**
```js
// 변경 전
accumulated: { gold: 0, material: 0, items: [] }

// 변경 후
accumulated: { gold: 0, materials: { common: 0, advanced: 0, rare: 0, legendary: 0 } }
// items 배열 제거 (장비 드롭 완전 제거)
```

**장비 드롭 코드 제거:**  
`initDispatchCombat()` 내 장비 드롭 블록 전체 제거 (EQUIP_DROP_CHANCE 로직).

**재료 드롭 등급 분리 — 확정 비율:**

| 단계 범위 | 일반 | 고급 | 희귀 | 전설 |
|----------|------|------|------|------|
| 1~10단계 | 97% | 3% | 0% | 0% |
| 11~20단계 | 15% | 83% | 2% | 0% |
| 21~30단계 | 3% | 12% | 83% | 2% |
| 31~40단계 | 1% | 2% | 10% | 87% |

- 10단계 이하에서 희귀·전설 재료 드롭 없음
- 고급 재료는 1~10단계에서 3% 극소량만 — 초반 플레이어가 우연히 마법 책 1~2권 만들 여지는 있지만 스팸 불가
- 각 구간에서 낮은 등급 재료가 소량 혼합 드롭되어 합성 원료는 꾸준히 축적

기존 `materialPerMin` 총량은 유지하되 위 비율로 등급 분배.  
오프라인 계산(`calcOfflineRewards`)도 동일 비율 적용.

## 3단계: 건물 업그레이드 시간 소요

### 3-A. State 변경 (`js/state.js`)

**DEFAULT_STATE에 추가:**
```js
buildingUpgrade: null,  // { buildingId, targetLevel, finishAt }
```

**upgradeBuilding() 변경:**  
즉시 완료 → 타이머 시작.  
기존 기능은 유지, `State.buildingUpgrade = { buildingId, targetLevel, finishAt }` 설정.

**게임 루프에서 완료 감지:**  
`checkBuildingUpgradeComplete()` 함수 추가 — finishAt 도달 시 레벨 반영 + 알림.

---

### 3-B. 건물 업그레이드 소요 시간 (확정)

최대 48시간 상한, 초반 ×3~4 → 후반 ×2 수준으로 점진 완화되는 구조.

| 레벨 | 소요 시간(초) | 표시 | 이전 대비 배율 |
|------|-------------|------|-------------|
| Lv 1 | 15 | 15초 | — |
| Lv 2 | 60 | 1분 | ×4.0 |
| Lv 3 | 180 | 3분 | ×3.0 |
| Lv 4 | 600 | 10분 | ×3.3 |
| Lv 5 | 1800 | 30분 | ×3.0 |
| Lv 6 | 5400 | 1시간 30분 | ×3.0 |
| Lv 7 | 14400 | 4시간 | ×2.7 |
| Lv 8 | 36000 | 10시간 | ×2.5 |
| Lv 9 | 86400 | 24시간 | ×2.4 |
| Lv 10 | 172800 | 48시간 | ×2.0 |
| Lv 11~18 | 172800 | 48시간 | — (창고 전용, 상한 유지) |

- Lv 1~3: 수십 초 ~ 수 분, 부담 없음
- Lv 4~5: 10~30분 — 첫 체감 대기
- Lv 6~7: 1~4시간 — 하루 수 회 확인 패턴
- Lv 8~9: 10~24시간 — 하루 1~2회 확인 패턴
- Lv 10+: 48시간 — 엔드게임 목표로서의 무게감 (최대치)

모든 건물 Lv 10 완전 업그레이드 총 시간: 건물당 약 5.2일, 건물 5개 순차 진행 시 약 26일

---

### 3-C. 건물 업그레이드 재료 비용 구조 (`js/data.js`, `js/state.js`)

기존 `baseCost.material` 단일 값 → 레벨 범위별 재료 등급 매핑.

**BUILDINGS 구조 확장 제안:**
```js
matGradeByLevel: {
  '1-3': 'common',
  '4-6': 'advanced',
  '7-8': 'rare',
  '9-10': { rare: 0.5, legendary: 0.5 }  // 9~10은 희귀+전설 혼합
}
```

창고 보관 칸 공식: `lv * 5 + 20` (Lv1: 25칸, Lv10: 70칸, Lv18: 110칸)

구체적 수량은 기존 `baseCost.material × costMult` 공식을 등급별로 분리.  
예: 기존 재료 비용 10 → 일반 재료 10 (1~3레벨), 고급 재료 10 (4~6레벨) 식으로 전환.

> ⚠️ **밸런스 협의 필요**: 건물별 구체 수량 (본부, 대기실, 공방, 창고, 연구소 각각 다름).

## 리스크 및 주의사항

1. **기존 저장 데이터**: `materials` 숫자 → 오브젝트, `laboratory` → `workshop` 키 변경 모두 마이그레이션 필수. 누락 시 기존 유저 진행도 손실.
2. **장비 드롭 제거**: 인벤토리에 장비가 없는 초반 유저는 공방에서만 장비를 얻게 됨 → 초반 공방 접근성 확인 필요.
3. **파견 UI 표시**: `accumulated.material` 단일 수치 → 4종 표시로 바뀌어 레이아웃 공간 재검토 필요.
4. **건물 업그레이드 중 이탈**: 앱 종료 후 재접속 시 타이머가 정상 동작하는지 finishAt 기반으로 검증 필요.
