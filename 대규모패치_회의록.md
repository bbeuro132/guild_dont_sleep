# 대규모 패치 회의록 — 경제 구조 개편

> 확정일: 2026-06-21
> 상태: 확정 (구현 대기)

---

## 배경

현재 경제 구조의 근본 문제: **파견만 보내면 전투 승패와 무관하게 패시브 수입(goldPerSec, materialPerMin)이 무한 축적**됨.
이로 인해 골드/재료의 가치가 사라지고, 건물 업그레이드·제작 등 모든 소모처가 시간 병목에만 의존하는 구조.

---

## 확정 사항

### 1. 수입 구조 전환: 패시브 → 전투 승리 기반

**핵심 변경**: 재화 수급의 주 수입원을 패시브(시간 비례)에서 **전투 승리 보상**으로 전환.

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 패시브 수입 (goldPerSec) | 100% | **15%로 축소** (최소 안전망) |
| 전투 승리 보상 | 전체 수입의 ~8% | **전체 수입의 ~85%** |
| 승률 0% 시 수입 | 패시브 100% 그대로 | 패시브 15%만 (현재의 약 1/7) |

**몬스터별 드롭 시스템:**
- 각 몬스터에 `gold`, `material` 드롭량 지정
- 기본값 자동 산출 (개별 오버라이드 가능):
  ```js
  gold     = monster.gold     ?? (monster.boss ? stage * 15 : stage * 3)
  material = monster.material ?? (monster.boss ? stage * 0.5 : stage * 0.1)
  ```
- 전투 승리 시 처치한 몬스터의 드롭 합산이 수입
- 패배 시 드롭 없음 (패시브만 유지)

**오프라인 계산:**
- 지역 몬스터들의 **평균 드롭량** 사용
- 보스 조우 비율 약 20% 가정
  ```js
  offlineGoldPerWin = avgNormalGold * 0.8 + avgBossGold * 0.2
  ```
- 총 전투 수 × 승률 × 평균 드롭 = 오프라인 수입

**영향 범위:**
- `js/state.js`: `tickDispatches()` 패시브 비율 축소, `processOfflineProgress()` 평균 드롭 계산
- `js/combat.js`: 전투 승리 시 몬스터별 드롭 적용
- `js/data.js`: 몬스터 데이터에 `gold`, `material` 필드 추가 (선택적), `goldPerSec`/`materialPerMin` 15%로 축소

### 2. 파티 전투력 공식 (확정 · 구현 완료)

```
개인 DPS   = ATK × (1 + CRIT/100 × CRITDMG/200) × (1 + SPD/100)
개인 생존력 = HP × (1 + DEF/50)

파티 DPS      = Σ 개인 DPS (치유사는 ×0.3)
파티 기본 생존 = Σ 개인 생존력
치유 기여     = Σ 치유사의 ATK × (1 + SPD/100) × 20  (치유계수 = 20)
파티 유효 생존 = 파티 기본 생존 + 치유 기여

파티 전투력   = 파티 DPS × √(파티 유효 생존)
```

- `calcPartyPower(team)`, `calcAreaPower(area, progress)` 함수로 구현 완료
- `estimateOfflineWinRate()`가 새 공식 기반으로 승률 추정
- `HEAL_COEFFICIENT = 20` 상수로 분리되어 조정 용이

### 3. 단계별 증가 배율 완만화

**현재**: goldPerSec이 단계마다 약 2~2.5배 증가 → 후반 기하급수적 인플레이션
**변경**: 전투 보상 기본값 공식 `stage * 3` (일반), `stage * 15` (보스)가 **선형 증가**이므로, 패시브 → 전투 전환 시 자연스럽게 배율이 완만해짐.

추가 조치가 필요하면 몬스터 드롭 공식의 stage 지수를 조정:
```js
// 선형 (현재 제안): stage * 3
// 약한 지수: stage * Math.pow(stage, 0.3) → 1단계 3, 10단계 60, 40단계 414
```

> 1차 배포 후 데이터를 관찰하여 추가 조정 여부 판단.

### 4. 장비 강화/리롤 시스템 (골드 싱크)

**목적**: 즉시 소모 가능한 반복적 골드 싱크 제공.

**설계:**
- 장비의 랜덤 옵션을 골드 + 재료를 소모하여 **리롤** (새 옵션으로 교체)
- 비용: 장비 등급에 비례
  ```
  일반: 500G + 일반 재료 5
  마법: 2,000G + 고급 재료 3
  희귀: 8,000G + 희귀 재료 2
  영웅: 25,000G + 희귀 재료 5
  전설: 80,000G + 전설 재료 3
  신화: 250,000G + 전설 재료 10
  ```
- 즉시 완료 (시간 소요 없음) → 골드를 쓸 기회가 시간에 의해 제한되지 않음
- 리롤 횟수 제한 없음 → 무한 반복 가능한 골드 싱크

**영향 범위:**
- `js/state.js`: `rerollEquipment(advId, slot)` 함수 추가
- `js/data.js`: 등급별 리롤 비용 테이블
- `js/ui.js`: 모험가 상세 장비 슬롯에 "리롤" 버튼 추가

---

### 5. 동시 진행 슬롯 해금 (시간 병목 완화 + 골드 싱크)

**현재**: 건물 업그레이드 1개, 공방 제작 1개만 동시 진행 가능
**변경**: 추가 슬롯을 **lifetimeGold 기반 비용**으로 해금

**슬롯 해금 비용:**
```
2번째 슬롯: lifetimeGold × 0.1%
3번째 슬롯: lifetimeGold × 0.3%
```

**특성:**
- 영구 해금 (리빌딩 후에도 유지)
- lifetimeGold는 단조증가 → 비용 변동 없음, 안정적
- 이미 프레스티지 시스템에서 사용 중인 값 재활용

**적용 대상:**
- 건물 업그레이드 슬롯 (현재 1개 → 최대 3개)
- 공방 제작/합성 슬롯 (현재 1개 → 최대 3개)

**영향 범위:**
- `js/state.js`: `DEFAULT_STATE`에 `extraBuildSlots`, `extraLabSlots` 추가, 해금 함수
- `js/ui.js`: 길드 탭·공방 탭에 슬롯 해금 UI, 동시 큐 관리
- `js/main.js`: 복수 큐 틱 처리

### 6. 재료 합성 시간 단축

**현재:**
| 합성 | 시간 |
|------|------|
| 일반 10 → 고급 1 | 300초 (5분) |
| 고급 10 → 희귀 1 | 900초 (15분) |
| 희귀 10 → 전설 1 | 2,700초 (45분) |

**변경:**
| 합성 | 시간 | 비고 |
|------|------|------|
| 일반 10 → 고급 1 | **60초 (1분)** | 5배 단축 |
| 고급 10 → 희귀 1 | **180초 (3분)** | 5배 단축 |
| 희귀 10 → 전설 1 | **600초 (10분)** | 4.5배 단축 |

> 공방 레벨 속도 보너스(Lv당 +10%)는 그대로 적용.

**합성에 골드 비용 추가** (추가 골드 싱크):
| 합성 | 골드 비용 |
|------|----------|
| 일반 → 고급 | 500G |
| 고급 → 희귀 | 3,000G |
| 희귀 → 전설 | 15,000G |

**영향 범위:**
- `js/data.js`: `SYNTHESIS_RECIPES`의 `craftTime` 수정, `gold` 필드 추가
- `js/state.js`: `startSynthesis()`에 골드 차감 로직 추가

## 배포 계획

### 1차 배포 (동시 적용)
| 항목 | 우선순위 |
|------|---------|
| 패시브 → 전투 승리 기반 전환 | 0순위 (근본 원인 제거) |
| 단계별 배율 완만화 | 1순위 (전환 시 자연 흡수) |
| 장비 리롤 시스템 | 2순위 (골드 싱크) |
| 동시 진행 슬롯 해금 | 2순위 (시간 병목 + 골드 싱크) |
| 재료 합성 시간 단축 + 골드 비용 | 2순위 (QoL + 골드 싱크) |

### 2차 (관찰 후 미세조정)
- 수급량 자체 축소 여부
- 오프라인 효율 감소 여부
- 몬스터 드롭 공식 지수 조정
- 합성 비율(10:1) 조정 여부
