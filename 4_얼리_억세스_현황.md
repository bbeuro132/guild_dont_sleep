# 길드는 잠들지 않는다 — 얼리 액세스 현황

> 작성일: 2026-06-16  
> 총 커밋: 54회  
> 기술 스택: 바닐라 JS (프레임워크 없음), HTML/CSS  
> 저장소: https://github.com/bbeuro132/guild_dont_sleep

---

## 게임 개요

플레이어가 길드를 운영하는 **방치형 RPG**. 모험가를 모집하고 각 지역에 파견해 전투를 반복하며 골드·재료를 수집, 건물과 모험가를 성장시키는 루프로 구성. 탭을 닫아도 오프라인 진행이 보상으로 정산된다.

## 구현 완료 기능

### 1. 전투 시스템

| 항목 | 내용 |
|------|------|
| 백그라운드 전투 | `BattleEngine` — 탭이 활성화된 상태에서 6초마다 즉시 시뮬레이션 |
| 실시간 관람 | `LiveBattle` — 팝업에서 1.5초/액션 속도로 재생, 로그 표시 |
| 피해 공식 | 물리: `ATK × (100 / (100 + DEF))`, 마법: 방어 무시 별도 계수 |
| 치명타 | 확률·배율 적용, 직업별 스킬에서 치명타 가능 여부 분기 |
| 상태이상 | 독 (매 턴 ATK 비례 피해), 출혈, 기절, 표식(사냥꾼 계열) |
| 보스 | 지역별 2종 보스 등장, 진행도 비례 출현 확률 상승 |
| 전멸 패널티 | 진행도 50% 감소 (리빌딩 노드로 최대 75% 보존까지 완화 가능) |

### 2. 직업·전직 시스템

3차 전직까지 완전 구현 (4·5차는 설계 완료, 미구현).

```
전사 계열:   전사 → 기사 / 검투사 → 수호기사 / 챔피언
도적 계열:   도적 → 암살자 / 사냥꾼 → 닌자 / 현상금 사냥꾼
마법사 계열: 마법사 → 대성당 술사 / 상아탑 술사 → 대성당 학자 / 상아탑 현자
치유사 계열: 치유사 → 성직자 / 용사제 → 신관 / 심문관
```

- 2차 전직: Lv.10 + 골드 2,000 + 재료 10
- 3차 전직: Lv.25 + 골드 8,000 + 재료 30 + 리빌딩 필요
- 모험가 상세 패널에서 보유 스킬 전체 확인 가능

### 3. 직업별 스킬

스킬 유형은 두 가지:
- `cooldown` — 전투 시작 시 쿨다운 최대값으로 초기화, 이후 턴마다 감소해 발동
- `passive` — 전투 시작 즉시 자동 적용, 발동 없이 전투 내내 유지

### 4. 파견 지역

**40개 지역**, 1단계(베른 근교 숲)부터 40단계까지 순차 해금. 국가 대분류 구조:

| 국가 | 단계 범위 |
|------|-----------|
| 베른 | 1 ~ 5 |
| 칸 | 6 ~ 10 |
| 솔 | 11 ~ 15 |
| 에테리아 | 16 ~ 20 |
| 오르도스 | 21 ~ 25 |
| 아이언피스트 | 26 ~ 30 |
| 드래곤 왕국 | 31 ~ 35 |
| 미지의 영역 | 36 ~ 40 |

- 지역별 진행도 200 달성 시 최대 진행도 기록 후 초기화(무한 반복)
- 진행도에 따라 보스 출현 확률 상승

### 5. 오프라인 보상

탭 종료 후 복귀 시 경과 시간을 정산.

- 파티 전투력 기반 승률 추정 → 추정 전투 횟수 계산
- 골드 / 재료 / 경험치 오프라인 정산
- 장비 드롭: 일반 전투 1%, 보스 전투 5% 확률로 드롭, 단계별 등급 풀 적용
- 복귀 시 보상 팝업에 상세 내역(장비 목록 포함) 표시

### 6. 건물 시스템

5종 건물, 레벨 업에 따라 기능 확장.

| 건물 | 기능 |
|------|------|
| 지휘 본부 | 최대 파견 수 증가 |
| 대기실 | 최대 모험가 수 증가 |
| 모집 서류 | 모집 슬롯 수 증가, 15분 자동 갱신 |
| 창고 | 인벤토리 용량 확장 |
| 연구소 | 경험치 서 제작 해금 |

### 7. 장비 시스템

- **6등급 체계**: 일반 → 마법 → 희귀 → 영웅 → 전설 → 신화
- **슬롯**: 무기(계열별 전용) / 방어구 / 악세서리
- **랜덤 옵션**: 2~3개 부여, 전투 옵션(방어 관통, 흡혈, 보호막 등) + 파견 옵션(골드 보너스, 재료 보너스 등)
- 전투·파견 중 드롭, 인벤토리 관리 및 장착 UI 구현

### 10. 리빌딩 (프레스티지) 시스템

조건 달성 후 길드를 초기화하는 대신 **영구 보너스(경지 포인트)** 획득.

**리빌딩 조건:**
- 1~5단계 지역 진행도 200 각각 달성
- 지휘 본부 Lv.5 이상

**경지 트리 (4가지 가지, 총 14노드):**

| 가지 | 효과 |
|------|------|
| 성장 | 경험치 +60%, 골드·재료 +60% |
| 모험 | 파견 시작 진행도 +50, 전투 진행도 +1, 보스 진행도 +3 |
| 생존 | 최대 HP +50%, 방어력 +35%, 전멸 진행도 보존 75% |
| 경지 | 4차 전직 / 5차 전직 해금 (전용 노드) |

## 버그 수정 이력 (주요)

| 수정 내용 | 영향 |
|-----------|------|
| 물리 피해 공식 비율식 변경 | 방어력이 클수록 무한 방어 불가, 균형 개선 |
| 전투 쿨다운 즉발 버그 | 전투 시작 직후 스킬 즉시 사용 방지 |
| 경험치 서 스택 전체 적용 | 동일 이름 경험치 서를 1개씩이 아닌 보유량 전부 한 번에 적용 |
| 레벨업 파견 HP 비율 동기화 | 레벨업으로 최대 HP 증가 시 파견 중 HP를 비율대로 유지 |
| 사냥꾼 표식 메커니즘 수정 | 표식 대상 사망 시 새 대상으로 자동 전환 |
| 전투 팝업 viewerActive 누출 | 다른 지역 팝업 전환 시 이전 지역 배경 전투 영구 정지되던 버그 |
| 좌표 붕괴 사망 로그 누락 | 보호막 무시 스킬로 처치 시 배틀로그에 사망 메시지 누락 |
| 빛의 장막·빛의 축복 패시브 전환 | cooldown 방식으로 잘못 구현되어 있던 패시브 스킬을 전투 시작 즉시 적용되도록 수정 |

## 기술 구조 요약

```
index.html
js/
  main.js   — 초기화, 게임 루프 (1초 tick), 저장/불러오기
  state.js  — 전역 State 객체, giveExp, tickDispatches, 오프라인 정산
  data.js   — JOBS, AREAS, SKILLS, BUILDINGS, SHOP, 리빌딩 조건
  combat.js — BattleEngine, LiveBattle, CombatUnit, 몬스터 분류
  ui.js     — 모든 화면 렌더링, 이벤트 핸들러, 팝업 관리
```

- 브라우저 `localStorage`에 전체 State를 JSON 직렬화해 자동 저장
- 외부 라이브러리·빌드 도구 없음 (순수 HTML+JS+CSS)
